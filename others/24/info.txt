Question 24: Create a docker image of simple login form using Django on port 8006

================================================================================
WHAT IS DJANGO?
================================================================================

Django is a high-level Python web framework that enables rapid development of 
secure and maintainable websites. It's FREE and OPEN SOURCE.

Key Features:
- Built-in authentication system (what we're using here!)
- Admin interface (automatically generated)
- ORM (Object-Relational Mapping) for database
- Template engine for HTML
- URL routing
- Security features (CSRF protection, SQL injection prevention)

Think of it like: Flask is a microframework (minimal), Django is a full-stack 
framework (batteries included).

HOW IT WORKS - STEP BY STEP
================================================================================

1. USER VISITS http://localhost:8006/
   → Django checks urls.py
   → Redirects to /login/

2. USER SEES LOGIN PAGE
   → views.py → login_view() function runs
   → Renders login.html template
   → Shows Bootstrap-styled form

3. USER ENTERS CREDENTIALS (admin/admin123)
   → Form submits via POST request
   → Django's authenticate() checks username/password
   → If valid: creates session, redirects to /dashboard/
   → If invalid: shows error message

4. USER SEES DASHBOARD
   → views.py → dashboard() function runs
   → Checks if user is logged in (session exists)
   → Renders dashboard.html with username

5. USER CLICKS LOGOUT
   → views.py → logout_view() function runs
   → Django destroys session
   → Redirects back to /login/

================================================================================
DJANGO AUTHENTICATION SYSTEM
================================================================================

Django provides built-in functions we're using:

1. authenticate(username, password)
   - Checks if credentials are valid
   - Returns User object if valid, None if invalid

2. login(request, user)
   - Creates a session for the user
   - Stores session ID in cookie

3. logout(request)
   - Destroys the session
   - Clears the cookie

4. @login_required decorator
   - Protects pages that need authentication
   - Redirects to login if not authenticated

5. User model
   - Built-in database table for users
   - Fields: username, password, email, etc.
   - Password is automatically hashed (secure!)

================================================================================
DOCKER CONTAINERIZATION
================================================================================

Dockerfile does:
1. FROM python:3.9-slim        ← Base image with Python
2. COPY requirements.txt       ← Copy dependencies list
3. RUN pip install Django      ← Install Django
4. COPY all project files      ← Copy our code
5. RUN python manage.py migrate ← Create database tables
6. RUN create superuser        ← Create admin account
7. CMD runserver 0.0.0.0:8006  ← Start Django server

Port 8006 is mapped: host:8006 → container:8006

================================================================================
SECURITY FEATURES INCLUDED
================================================================================

1. CSRF Protection
   - {% csrf_token %} in forms prevents cross-site attacks

2. Password Hashing
   - Passwords stored as hashes, not plain text
   - Uses PBKDF2 algorithm with SHA256

3. SQL Injection Prevention
   - Django ORM automatically escapes queries

4. XSS Protection
   - Templates auto-escape HTML

5. Session Security
   - Secure session cookies
   - Session expiry

FOR LAB EXPLANATION
================================================================================

When explaining to examiner, say:

"This is a Django web application with a custom login system. Django is a 
Python web framework that provides built-in authentication. I created custom 
HTML templates for the login and dashboard pages, styled with Bootstrap. 

The application uses Django's User model for storing accounts in SQLite 
database. When a user logs in, Django's authenticate() function verifies 
credentials and creates a session. The session is stored in cookies.

I containerized it with Docker - the Dockerfile installs Django, copies the 
project files, runs database migrations, creates a superuser, and starts the 
development server on port 8006.

The project has two main parts: 'loginproject' for configuration and 
'loginapp' for the login functionality. This follows Django's project/app 
architecture where one project can have multiple apps."
