Docker Volume Mount Demo

Bind Mount

Maps host directory directly to container

-v /host/path:/container/path
  • Direct file access
  • Real-time sync
  • Development friendly

Named Volume

Docker-managed storage

-v volume-name:/container/path
  • Managed by Docker
  • Better performance
  • Production ready

Example Commands

Create and mount directory:
docker run -v $(pwd)/data:/app/data nginx:alpine
Read-only mount:
docker run -v $(pwd)/data:/app/data:ro nginx:alpine
Named volume:
docker run -v mydata:/app/data nginx:alpine
